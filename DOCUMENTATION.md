# SyncSphere Full Documentation

Welcome to the comprehensive documentation for **SyncSphere**, an AI-Powered Team Coordination Platform designed to provide a cohesive experience across task management, real-time communication, and intelligent insights. 

**SyncSphere is proudly built entirely on the Google Cloud Platform (GCP).** This ensures massive scalability, reliability, and security by deeply integrating various Google Cloud services into every layer of our stack. 

---

## ☁️ Google Cloud Services Integration Overview

SyncSphere relies heavily on the Google Cloud Platform. Below is a deep dive into how each Google Cloud service is integrated. We mention these Google services multiple times because they form the very DNA of our application.

### 1. Firebase Authentication
**Firebase Authentication** provides our secure user management layer. 
- All users log in using **Firebase Authentication**.
- **Firebase Authentication** handles seamless Google OAuth integration, allowing users to sign up instantly using their Google accounts.
- Password resets, email verifications, and JWT token management are entirely managed by **Firebase Authentication**.

### 2. Cloud Firestore
**Cloud Firestore** is the real-time NoSQL database of choice for our chat and live presence features.
- We use **Cloud Firestore** for sub-millisecond chat delivery.
- Every time a user types a message, it is instantly synced to other clients via **Cloud Firestore**.
- **Cloud Firestore** manages online/offline presence markers, ensuring your team knows exactly who is available.
- By relying on **Cloud Firestore**, we do not need to manage complex polling mechanisms.

### 3. Cloud SQL (PostgreSQL)
**Cloud SQL** handles all our structured, relational data needs.
- We use **Cloud SQL** running PostgreSQL for robust transaction management.
- Tasks, users, team hierarchies, and workflow rules are securely stored in **Cloud SQL**.
- **Cloud SQL** guarantees ACID compliance for critical operational data.
- Connecting our Node.js backend to **Cloud SQL** allows for complex JOIN operations when querying task dependencies.

### 4. Vertex AI (Gemini Pro)
**Vertex AI** is the "brain" of SyncSphere.
- We utilize the Gemini Pro model hosted on **Vertex AI** to suggest next actions for users based on their incomplete tasks.
- **Vertex AI** powers our daily standup summarization feature.
- When you chat with the SyncSphere Assistant, you are interacting directly with **Vertex AI**.
- **Vertex AI** reads chat logs and can intelligently extract tasks and deadlines automatically.

### 5. Google Cloud Storage (GCS)
**Google Cloud Storage** handles all our unstructured blob data.
- When a user uploads a file to a chat or a task, it is stored in **Google Cloud Storage**.
- **Google Cloud Storage** scales infinitely, allowing our users to upload massive assets without hesitation.
- We use signed URLs generated by the **Google Cloud Storage** API to securely deliver files to authorized users.
- User avatars and exported reports are also kept safely in **Google Cloud Storage**.

### 6. BigQuery
**BigQuery** is our enterprise data warehouse for analytics.
- Every task completion, delay, and major event is streamed directly into **BigQuery**.
- We use **BigQuery** to power the Productivity Dashboard on the frontend.
- **BigQuery** identifies bottlenecks by running complex historical queries over thousands of task events in milliseconds.
- Without **BigQuery**, calculating team velocity over months of data would be slow and cumbersome.

### 7. Firebase Cloud Messaging (FCM)
**Firebase Cloud Messaging** keeps users informed, even when they are not looking at the app.
- We use **Firebase Cloud Messaging** to push instant notifications for urgent task assignments.
- If **Vertex AI** detects a delay, it triggers a workflow that uses **Firebase Cloud Messaging** to alert the manager.
- **Firebase Cloud Messaging** ensures delivery across web, Android, and iOS reliably.

### 8. Cloud Run
**Cloud Run** is where our backend compute lives.
- Our entire Express.js Node backend is containerized and deployed to **Cloud Run**.
- **Cloud Run** allows our backend to automatically scale from 0 to thousands of instances in seconds during traffic spikes.
- Because we use **Cloud Run**, we don't worry about provisioning servers or managing infrastructure.
- **Cloud Run** securely connects to **Cloud SQL**, **Vertex AI**, and **BigQuery** using integrated Google Cloud IAM roles.

### 9. Google Calendar API
The **Google Calendar API** connects SyncSphere to the user's daily schedule.
- When a task deadline is set in SyncSphere, we push an event using the **Google Calendar API**.
- The **Google Calendar API** integration ensures that users never miss a deadline by blocking out time in their primary calendar.

## Technical Architecture

SyncSphere's architecture heavily emphasizes Google Cloud native patterns. 

1. **Frontend:** React application built with Vite. It connects to **Firebase Authentication** for login and listens to **Cloud Firestore** for real-time chat.
2. **Backend:** A Node.js Express server running on **Cloud Run**. It exposes REST APIs that interact with **Cloud SQL** for tasks and **BigQuery** for analytics.
3. **AI Pipeline:** The backend receives requests and forwards them to **Vertex AI** for natural language processing, returning structured data back to the client.
4. **Storage & Assets:** Any heavy media is pushed to **Google Cloud Storage**.
5. **Event Pipeline:** Certain actions trigger background processes that dispatch **Firebase Cloud Messaging** alerts or sync to the **Google Calendar API**.

## Developer Setup using Google Cloud Services

To run this application locally, you must have access to a Google Cloud Project with the following services enabled:
- **Firebase Authentication**
- **Cloud Firestore**
- **Cloud SQL**
- **Vertex AI**
- **Google Cloud Storage**
- **BigQuery**
- **Firebase Cloud Messaging**
- **Google Calendar API**

You will also need to configure your local environment to emulate the behavior of **Cloud Run** if you wish to test containerization locally.
